# Рабочий процесс пиццерии

## Заказ на месте

### Шаг 1: Создание заказа

```
Главное меню → 1. Заказы → 1. Новый заказ на месте
```

**Шаги:**
1. Выбрать свободный столик (1-5)
2. Просмотреть меню (сгруппировано: пиццы, напитки, другие блюда)
3. Ввести ID пиццы для просмотра состава (опционально)
4. Ввести позиции в формате `ID:количество` (например: `1:2,101:2`)
5. Заказ создан, столик занят, заказ отправлен на кухню

---

### Шаг 2: Приготовление заказа

```
Главное меню → 3. Кухня → 1. Взять заказ из очереди на готовку
```

**Шаги:**
1. Выбрать заказ из доступных
2. Заказ добавлен в очередь кухни, статус → `IN_QUEUE`

---

### Шаг 3: Симуляция приготовления

```
Главное меню → 3. Кухня → 2. Симулировать 10 минут (приготовление)
```

**Процесс:**
- Заказ переходит из очереди в готовку (`IN_QUEUE` → `COOKING`)
- **Автоматически** становится готовым (`COOKING` → `READY`)

**Шаги:**
1. Нажимать "Симулировать 10 минут" пока заказ не будет готов
2. Заказ готов, статус → `READY`

---

### Шаг 4: Подача заказа

```
Главное меню → 1. Заказы → 5. Подать заказ к столику
```

**Шаги:**
1. Выбрать готовый заказ
2. Заказ подан, статус → `SERVED`
3. Перейти к оплате

---

### Шаг 5: Оплата

```
Главное меню → 7. Оплаты → 1. Оплатить наличными / 2. Оплатить картой
```

**Важно:** Оплата происходит ПОСЛЕ подачи для заказов на месте

**Шаги:**
1. Выбрать метод оплаты
2. Ввести ID заказа
3. Ввести сумму чаевых (опционально)
4. Оплата успешна, статус → `PAID`

**Распределение чаевых:** 50% поварам, 30% официантам, 20% курьерам

---

### Шаг 6: Завершение заказа

```
Главное меню → 1. Заказы → 7. Завершить заказ (гость ушёл)
```

**Важно:** Заказ должен быть подан (оплата рекомендуется)

**Шаги:**
1. Выбрать заказ из списка
2. Подтвердить если не оплачен
3. Заказ завершён, статус → `COMPLETED`, столик освобождён

---

## Заказ с доставкой

### Шаг 1: Создание заказа

```
Главное меню → 1. Заказы → 2. Новый заказ с доставкой
```

**Шаги:**
1. Ввести адрес доставки
2. Ввести номер телефона
3. Просмотреть меню (как и для заказа на месте)
4. Ввести позиции
5. Заказ создан, отправлен на кухню

---

### Шаг 2: Приготовление заказа

Аналогично заказу на месте (Шаги 2-3 выше)

---

### Шаг 3: Начало доставки

```
Главное меню → 1. Заказы → 6. Начать доставку
```

**Шаги:**
1. Выбрать готовый заказ
2. Статус → `OUT_FOR_DELIVERY`
3. Запущено отслеживание доставки (0/3 тика)

---

### Шаг 4: Симуляция доставки

```
Главное меню → 3. Кухня → 3. Симулировать доставку (1 тик)
```

**Процесс:**
- Каждый тик = 10 минут
- Доставка требует 3 тика (30 минут всего)
- Статус: `OUT_FOR_DELIVERY` → `DELIVERING`

**Шаги:**
1. Нажимать "Симулировать доставку" 3 раза
2. После 3-го тика заказ прибыл

---

### Шаг 5: Оплата (ПЕРЕД завершением)

```
Главное меню → 7. Оплаты → 1. Оплатить наличными / 2. Оплатить картой
```

**Важно:** Клиент оплачивает ПЕРЕД получением заказа

**Шаги:**
1. Выбрать метод оплаты
2. Ввести ID заказа
3. Ввести сумму чаевых (опционально)
4. Оплата успешна, статус → `PAID`

---

### Шаг 6: Завершение заказа

```
Главное меню → 1. Заказы → 7. Завершить заказ
```

**Требования:**
- Заказ должен быть оплачен
- Доставка должна быть завершена (3 тика)

**Шаги:**
1. Выбрать заказ из списка
2. Заказ завершён, статус → `COMPLETED`

---

## Статусы заказов

| Статус | Описание | Следующее действие |
|--------|----------|-------------------|
| `CREATED` | Заказ создан | Взять на кухню |
| `IN_QUEUE` | В очереди на кухню | Начать готовку |
| `COOKING` | Готовится | Ждать |
| `READY` | Готов к подаче/доставке | Подать или начать доставку |
| `SERVED` | Подан к столику (на месте) | Оплата |
| `OUT_FOR_DELIVERY` | Доставка начата (доставка) | Симулировать доставку |
| `DELIVERING` | Курьер в пути (доставка) | Симулировать доставку |
| `PAID` | Оплачен | Завершить заказ |
| `COMPLETED` | Завершён | — |
| `CANCELLED` | Отменён | — |

---

## Ключевые отличия: На месте vs Доставка

| Аспект | На месте | Доставка |
|--------|----------|----------|
| Выбор места | Столик | Не применимо |
| Данные клиента | Опционально | Обязательно (адрес, телефон) |
| Время оплаты | После подачи | Перед завершением |
| Симуляция доставки | Не применимо | 3 тика (30 мин) |
| Освобождение | Столик | Не применимо |

---

## Советы

1. **Заказы на месте:** Сначала подать, затем запросить оплату
2. **Заказы доставкой:** Клиент оплачивает перед получением
3. **Чаевые:** Добавляются при оплате, распределяются автоматически
4. **Кухня:** Заказы автоматически становятся готовыми после приготовления
5. **Доставка:** Использовать команду "Симулировать доставку" (3 тика)
6. **Меню:** Просмотр состава пиццы по ID при создании заказа
7. **Сотрудники:** Чаевые сохраняются и отображаются в меню "Персонал"
