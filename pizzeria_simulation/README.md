# Пиццерия Симулятор

Симулятор управления пиццерией с полным циклом: заказы, кухня, доставка, платежи и персонал.

## Быстрый старт

```bash
# Запуск приложения
.venv/bin/python main.py

# Тесты
.venv/bin/pytest tests/ -v

# Проверка моделей
.venv/bin/python test.py
```

## Возможности

### Управление заказами
- Заказы на месте и с доставкой
- Оплата после подачи (на месте) или перед завершением (доставка)
- Чаевые с автоматическим распределением (50%/30%/20%)
- Кухня с симуляцией времени
- Управление столиками
- Отслеживание доставки (3 тика = 30 минут)

### Зал
- 5 столиков (2-8 мест каждый)
- Все столики свободны по умолчанию
- Отслеживание занятости

### Кухня
- Очередь заказов
- Автоматический переход в статус "готов"
- Симуляция времени

### Оплаты
- Наличные и карты
- Поддержка чаевых
- Финансовые отчёты

### Персонал
- Повара, официанты, курьеры, кассиры
- Учёт зарплат и чаевых

## Рабочий процесс

### Заказ на месте

```
1. Заказы → 1. Новый заказ на месте
   → Выбрать столик → Ввести позиции → Заказ создан

2. Кухня → 1. Взять заказ на готовку
   → Выбрать заказ → В очереди

3. Кухня → 2. Симулировать 10 минут
   → Повторить 1-2 раза
   → Заказ готов (автоматически!)

4. Заказы → 5. Подать заказ
   → Выбрать заказ → Подан к столику

5. Оплаты → 1/2. Оплатить
   → Ввести ID заказа + чаевые
   → Заказ оплачен

6. Заказы → 7. Завершить заказ
   → Гость ушёл → Столик свободен
```

### Заказ доставкой

```
1. Заказы → 2. Новый заказ с доставкой
   → Ввести адрес и телефон → Ввести позиции

2. Кухня → 1. Взять заказ на готовку
3. Кухня → 2. Симулировать 10 минут (пока не готов)

4. Заказы → 6. Начать доставку
   → Выбрать заказ → Доставка началась (0/3 тика)

5. Кухня → 2. Симулировать 10 минут
   → Повторить 3 раза (30 минут всего)
   → Заказ прибыл

6. Оплаты → 1/2. Оплатить
   → Клиент оплачивает ПЕРЕД получением

7. Заказы → 7. Завершить заказ
   → Заказ доставлен
```

## Статусы заказов

```
На месте:
CREATED → IN_QUEUE → COOKING → READY → SERVED → PAID → COMPLETED

Доставка:
CREATED → IN_QUEUE → COOKING → READY → OUT_FOR_DELIVERY → DELIVERING → PAID → COMPLETED
                                    (тик 0)      (тик 1-3)
```

## Ключевые правила

1. **Оплата на месте:** После подачи, перед завершением
2. **Оплата доставки:** Перед завершением (клиент платит при получении)
3. **Время доставки:** 3 тика = 30 минут
4. **Столик освобождается:** Только после завершения заказа
5. **Чаевые:** Автораспределение (50% поварам, 30% официантам, 20% курьерам)
